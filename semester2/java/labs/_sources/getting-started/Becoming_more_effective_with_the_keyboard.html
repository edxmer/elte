Becoming more effective with the keyboard in 9+1 easy steps

Introduction

When working in VS Code (or VSCodium), the keyboard is more than just an input device—it is your fastest path to flow. Using it well means you can spend less time reaching for the mouse, and more time thinking about your code. In this short journey, you will discover how the keyboard alone can guide you through the editor.

Part 1a

To begin, from the menu bar, choose File → Open Folder… and select the current folder. Inside, you will find several .java files in some subfolders. With the folder open, we are ready to practice.

  • Start each class by opening the top level work folder with File → Open Folder….
  • Recommended: for each class, create a subfolder in it, and work in there to keep your files neat and organized.

Try it now: open the folder that contains the demo .java files and confirm you can see them in the Explorer.

Part 1b

Enter Settings: click the bottom left ⚙️ (cogwheel) symbol, then choose Settings. There, type compact in the search bar. Uncheck the Explorer: Compact Folders button.

  • This shows you the folder structure in an easier-to-understand fashion.
  • You only have to do this once.

Try it now: maybe you won’t see a big change just yet, but you just wait until Part 8.

Part 2

Let’s start with movement. The arrow keys move your cursor one step at a time, but holding Ctrl while pressing arrows lets you jump word by word. This is faster than tapping the right arrow ten times. To reach the start or end of a line, try Home and End. With a little practice, you’ll begin to think in words and lines, not just letters.

  • Use arrow keys for single steps.
  • Hold Ctrl with arrows to jump word by word.
  • Use Home and End to reach start or end of a line.

Try it now: open one of the Java files, move to a long line, and practise jumping with Ctrl+Arrow, then Home and End.

Part 3

Once movement feels natural, add selections. Hold Shift with your arrows to select text. Combine it with Ctrl to select entire words at a time. When editing Java, this makes renaming identifiers or replacing arguments far quicker than dragging the mouse. Try moving the cursor into a method call and selecting just the method name with Ctrl+Shift+Right, then type something new.

  • Hold Shift with arrows to select text.
  • Combine Ctrl+Shift with arrows to select words quickly.

Try it now: in a Java file, select a method name using Ctrl+Shift+Arrow, then replace it with new text.

Part 4

Selections have special tricks that shine when writing Java code. Place your cursor on a symbol and press Ctrl+D to select the next word with the same text. This is useful for renaming: highlight a variable name, repeat Ctrl+D a few times, and rename it everywhere at once. Just be careful: a short name can match inside of a longer word!

Another favourite is Alt+Click, which adds multiple cursors. Try clicking in front of several IO.println statements, and you can edit them all at the same time.

  • Ctrl+D selects the next occurrence of a symbol.
  • Alt+Click adds multiple cursors for simultaneous editing.

Try it now: find a variable or repeated word, use Ctrl+D to select more occurrences, then use Alt+Click to edit several lines at once.

Part 5

Finding and opening files is essential in larger projects. Use Ctrl+P to open the quick file search. Start typing part of a filename, such as Main, and the editor will show matches immediately. Hit Enter to open one.

Also, you can use Ctrl+W to close the currently open file tab.

  • Ctrl+P opens quick file search.
  • Type part of a filename and press Enter to open it.
  • Ctrl+W closes the current file.

Try it now: press Ctrl+P, type the name of a Java file, and open it. Then press Ctrl+W and close it.

Part 6

Sometimes you need to see two files at once. With a file already open, press Ctrl+\ to split the editor. Now you can compare code side by side. Try opening Main.java on the left and a helper class on the right. To move focus between them, press Ctrl+1 and Ctrl+2. You’ll soon be comfortable editing across multiple files at once.

  • Ctrl+\ splits the editor into two.
  • Ctrl+1 and Ctrl+2 move focus between split groups.

Try it now: split the editor, open two different Java files, and switch focus between them with Ctrl+1 and Ctrl+2.

Part 7

Programming often involves the terminal (command line). Open it directly in the editor with Ctrl+` (backtick). You can run jshell and java and javac right here without leaving VS Code. To move focus between the terminal and the editor, use Ctrl+` again, or press Ctrl+1 to return to the editor. This keeps your hands on the keyboard while you switch seamlessly between editing and running your program.

  • Ctrl+` opens or toggles the terminal.
  • Use Ctrl+1 to focus the editor again.

Try it now: open the terminal, type a command, then switch back to the editor with Ctrl+1.

Part 8

In the terminal, you can use Tab (⭾) to autocomplete commands and filenames. Instead of having to type the name of a long folder’s name, you only type the first few letters, press Tab, and the whole name appears. If you have many filenames or folder names that can complete what you typed, you will get the first one—pressing Tab again gives you the next one.

  • Use Tab in the terminal for file and folder autocompletion.

Try it now: in the terminal, run mkdir -p follow\me\along\the\cavernous\journey. This makes a deep folder structure. Now type cd fo⭾ and you get cd .\follow\ but don’t press Enter yet; continue typing. In all, you only have to type cd fo⭾ me⭾ al⭾ the⭾ cav⭾ jo⭾ to get cd .\follow\me\along\the\cavernous\journey\.

Also, go back to Part 1b and toggle the option. See the difference?

Part 9

Finally, take advantage of recalling old commands in the terminal. Press the Up Arrow to cycle back through your last commands. This is especially useful when you are compiling the same file repeatedly. Edit Main.java, press Ctrl+` to jump down, hit the Up Arrow to recall your last Java related command, and run it again instantly. With this, you close the loop between coding and testing without breaking rhythm.

  • Use the Up Arrow in the terminal to recall previous commands.

Try it now: run a Java compile command in the terminal, then press the Up Arrow to repeat it after making a change.
